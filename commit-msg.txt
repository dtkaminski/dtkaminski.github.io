feat(connect): route Shopify connect through authenticated connect-start

The Shopify "Approve on Shopify" action was an <a href> navigation to the
public GET oauth-shopify-install?shop=..&brand=frkl — unauthenticated, brand
slug hardcoded, and spoofable via the &brand= param.

It now POSTs connect-start with the user's JWT (provider:'shopify', brand from
OI_BRAND.slug, shop), which verifies brand_users owner/admin membership, stamps
metadata.initiated_by_user_id, and returns the provider authorize_url — we then
send the top window to Shopify's consent screen. Endpoint base is derived from
OI_ASK.endpoint (no hardcoded project); auth mirrors the BusinessEconomicsPanel
save path (getOIAsk → ASK.getJwt).

- Removed the hardcoded SUPABASE_FNS / BRAND_SLUG and the buildInstallUrl GET.
- Modal button is disabled until a valid *.myshopify.com domain + an authed
  session; surfaces connect errors and a sign-in nudge.

COORDINATED RELEASE — deploy together with the staged fit-engine guards:
  oauth-shopify-callback (enforces initiated_by_user_id = owner/admin before it
  activates) and oauth-shopify-install (hardened / retire candidate). Deploying
  the callback guard without this change would 403 the old unstamped GET flow.

Rebuilt frkl-app.js via build-app.mjs.
